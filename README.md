# vsLearn

vsLearn is a VS Code-inspired web development learning platform. It presents courses, lessons, quizzes, coding challenges, progress tracking, and admin workflows inside an editor-style interface familiar to developers.

The current codebase is a frontend-first TanStack Start application with mock course and user data. The intended direction is a full-stack platform with authentication, persistent learning progress, course administration, code execution, certificates, and Cloudflare deployment.

## Tech Stack

- React 19
- TanStack Start
- TanStack Router
- TanStack Query
- TypeScript
- Vite
- Tailwind CSS v4
- shadcn/ui-style component primitives
- Radix UI
- Lucide React icons
- Cloudflare Workers tooling via Wrangler and Cloudflare Vite plugin
- ESLint and Prettier

## Features

- VS Code-inspired application shell with title bar, activity bar, sidebar, tabs, breadcrumbs, terminal panel, and status bar.
- Landing page for product positioning, features, categories, pricing, and testimonials.
- Course catalog with category filtering and search.
- Course detail pages with modules and lesson lists.
- Lesson viewer supporting text lessons, video placeholders, quizzes, and coding exercises.
- Browser-based coding challenge demo with output comparison.
- Dashboard with mock learner progress, badges, enrolled courses, and recommendations.
- Command palette for navigation and course discovery.
- Admin overview, course table, and user table mock interfaces.
- Branded root error page and SSR error handling wrapper.

## Planned Full-Stack Architecture

The project is designed to evolve into a Cloudflare-deployed full-stack application.

Planned architecture:

- TanStack Start for SSR, routing, loaders, and server functions.
- Cloudflare Workers as the runtime target.
- Cloudflare D1 for relational data such as users, courses, modules, lessons, enrollments, progress, attempts, certificates, and admin audit logs.
- Cloudflare R2 for assets such as lesson media, certificate PDFs, and uploaded course resources.
- Cloudflare KV or Durable Objects where low-latency state or coordination is useful.
- Server-side authentication with secure HTTP-only cookies.
- Role-based access control for students, instructors, and admins.
- Server-backed course loaders and mutations instead of static mock data.
- Sandboxed code execution for coding challenges with resource limits and deterministic test results.
- Certificate generation and verification pages.
- Admin dashboard backed by real analytics and moderation workflows.

## Setup Instructions

Prerequisites:

- Node.js 22 or newer recommended
- npm
- Wrangler CLI access if deploying to Cloudflare

Install dependencies:

```bash
npm install
```

Start the development server:

```bash
npm run dev
```

Build for production:

```bash
npm run build
```

Preview the production build locally:

```bash
npm run preview
```

## Environment Variables

The current mock-data version does not require environment variables to run locally.

Expected future variables and bindings:

```bash
SESSION_SECRET=
DATABASE_URL=
CLOUDFLARE_ACCOUNT_ID=
CLOUDFLARE_API_TOKEN=
R2_BUCKET_NAME=
```

For Cloudflare Workers development, local secrets should be placed in `.dev.vars`. Do not commit `.dev.vars` or other secret files.

Cloudflare D1/R2/KV bindings should be configured in `wrangler.jsonc` when those services are added.

## Contribution

Contributions should keep the existing product direction intact: a professional learning platform with a VS Code-inspired interface and pragmatic full-stack implementation.

Before opening a change:

- Run `npm run lint`.
- Run `npm run build` for functional changes.
- Keep changes focused and small where possible.
- Do not commit secrets, `.dev.vars`, generated Cloudflare state, or local build artifacts.
- Avoid editing `src/routeTree.gen.ts` manually; it is generated by TanStack Router.
- Prefer typed server loaders/functions over client-only mock data as the app moves toward production.
